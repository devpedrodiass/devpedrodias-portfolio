# Who is Pedro Dias?

Hey there! 👋 I'm Pedro Dias, a frontend developer who's been turning coffee into code for over 5 years now. My journey in web development has taken me from internships in Brazil to working with major international companies like FARFETCH, and I'm loving every minute of it.

## The Journey So Far

I started my career back in 2020 as an intern at **Pyxis Consult**, where I got my hands dirty with full-stack development. That experience taught me that while I could do backend, my heart truly belonged to the frontend—where design meets functionality and user experience becomes magic.

Fast forward to today, and I'm currently working as a **Frontend Developer at RD Saúde** (one of Brazil's largest pharmacy chains) while also contributing to **Quanthra AI** part-time. Yes, I like to keep myself busy! 😄

### Career Highlights

My career has been quite the adventure:

- **FARFETCH (2021-2024)**: Spent over 2 years working on luxury e-commerce at scale. Nothing teaches you performance optimization quite like handling millions of users browsing high-end fashion!

- **Descomplicando Linguagens (2023-2025)**: Built educational platforms that help people learn new languages. Turns out, making learning fun is just as challenging as making code performant.

- **Magik Tech (2024-2025)**: Wore the full-stack hat again, proving I can still handle backend when needed (but don't tell anyone I prefer React 😉).

## What I Actually Do

I specialize in building **scalable, high-performance web applications** using the JavaScript ecosystem. My weapons of choice?

- **React.js & Next.js**: For building lightning-fast, SEO-friendly applications
- **TypeScript**: Because `any` is not a type strategy
- **Node.js**: When I need to venture into backend territory
- **Modern CSS & Tailwind**: Making things look good is half the battle

I'm a big believer in **Clean Code** and **SOLID principles**. Sure, they might sound like buzzwords, but they're the difference between code that works and code that works _well_.

## My Development Philosophy

I believe in:

1. **User-First Design**: If users can't figure it out in 5 seconds, we've failed
2. **Performance Matters**: Nobody likes waiting for pages to load
3. **Code Quality**: Future-you will thank present-you for writing clean code
4. **Continuous Learning**: The JavaScript ecosystem changes faster than I can finish my coffee

## Working Style

I thrive in **agile environments** (Scrum/Kanban) and love collaborating with cross-functional teams. There's something special about designers, developers, and product managers coming together to build something awesome.

Version control? Git all the way. Proper branching strategies, meaningful commit messages, and code reviews that actually improve the codebase—that's my jam.
